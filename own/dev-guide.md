# 开发指南：AgentScope

## 环境搭建

### 系统要求

- **Python版本**：3.10 或更高
- **操作系统**：Linux / macOS / Windows（WSL推荐）
- **依赖管理**：pip 或 uv（推荐）

### 依赖安装步骤

#### 1. 基础安装

```bash
# 克隆仓库
git clone https://github.com/agentscope-ai/agentscope.git
cd agentscope

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或 venv\Scripts\activate  # Windows

# 安装基础依赖
pip install -e .

# 或使用uv（更快）
uv pip install -e .
```

#### 2. 完整开发环境（推荐）

```bash
# 安装所有开发依赖
pip install -e ".[dev]"

# 或指定特定功能
pip install -e ".[full]"      # 完整功能
pip install -e ".[rag]"       # RAG功能
pip install -e ".[memory]"    # 记忆功能
pip install -e ".[realtime]"  # 实时语音功能
pip install -e ".[tuner]"     # 微调功能
```

### 配置说明

#### 环境变量

创建`.env`文件或设置以下环境变量：

```bash
# 模型API密钥
export DASHSCOPE_API_KEY="your-dashscope-key"
export OPENAI_API_KEY="your-openai-key"
export ANTHROPIC_API_KEY="your-anthropic-key"
export GEMINI_API_KEY="your-gemini-key"

# AgentScope配置（可选）
export AGENTSCOPE_DISABLE_CONSOLE_OUTPUT="false"  # 禁用控制台输出
```

#### 预提交钩子（可选但推荐）

```bash
# 安装pre-commit钩子
pre-commit install

# 手动运行检查
pre-commit run --all-files
```

### 常用调试手段

#### 1. 打印Agent内部状态

```python
# 打印记忆内容
memory_msgs = await agent.memory.get_memory()
for msg in memory_msgs:
    print(f"{msg.name}: {msg.get_text_content()}")

# 查看工具集
print(agent.toolkit.get_json_schemas())
```

#### 2. 调试模型调用

```python

import logging
logging.basicConfig(level=logging.DEBUG)

print(prompt)
```

#### 3. 使用断点

```python
async def reply(self, msg):
    import pdb; pdb.set_trace()  # 设置断点
    # ... 后续代码
```

### 已知易踩的坑

1. **异步上下文问题**
   - AgentScope完全基于asyncio，确保在异步环境中调用
   - 使用`asyncio.run(main())`启动应用

2. **Formatter与Model不匹配**
   - 确保使用与模型对应的Formatter（如DashScopeChatModel配DashScopeChatFormatter）
   - 多智能体场景使用MultiAgentFormatter

3. **消息广播失效**
   - MsgHub需要在`async with`上下文中使用
   - 确保Agent继承自AgentBase以支持订阅机制

4. **工具函数签名**
   - 工具函数必须有类型注解和docstring
   - 异步工具函数需要返回`ToolResponse`

5. **流式输出处理**
   - 流式响应是AsyncGenerator，需要正确处理
   - 注意流式响应的累积逻辑

## 代码规范

### 命名约定

| 类型 | 命名风格 | 示例 |
|------|----------|------|
| 类 | PascalCase | `ReActAgent`, `ChatModelBase` |
| 函数/方法 | snake_case | `reply()`, `call_tool_function()` |
| 常量 | UPPER_SNAKE_CASE | `DEFAULT_MAX_ITERS` |
| 私有变量 | _snake_case | `_reply_task`, `_sys_prompt` |
| 模块 | 小写，下划线分隔 | `react_agent.py` |

### 目录组织约定

```
src/agentscope/
├── __init__.py           # 模块入口
├── _version.py           # 版本（私有模块前缀_）
├── agent/                # 功能模块
│   ├── __init__.py
│   ├── _agent_base.py    # 私有实现前缀_
│   └── react_agent.py
└── ...
```

### 文档字符串规范

使用Google风格docstring：

```python
async def reply(
    self,
    msg: Msg | list[Msg] | None = None,
) -> Msg:
    """Generate a reply based on the current state and input.

    Args:
        msg (`Msg | list[Msg] | None`, optional):
            The input message(s) to the agent.

    Returns:
        `Msg`: The output message generated by the agent.

    Raises:
        ValueError: If the input message format is invalid.
    """
```

### 提交信息格式

遵循Conventional Commits规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

类型：
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

示例：
```
feat(agent): add interrupt support for ReActAgent

- Add interrupt() method to AgentBase
- Handle CancelledError in reply()
- Add interruption message to memory

Closes #123
```

## 贡献流程

### 分支策略

- **main**: 稳定分支，只接受通过PR的代码
- **dev**: 开发分支，新功能先合并到dev
- **feature/***: 功能分支，从dev切出
- **fix/***: 修复分支，从dev切出

### PR / Code Review 流程

1. **Fork仓库**（外部贡献者）或**创建分支**（内部成员）

2. **创建功能分支**
   ```bash
   git checkout -b feature/my-feature
   ```

3. **提交代码**
   ```bash
   git add .
   git commit -m "feat(module): description"
   ```

4. **确保测试通过**
   ```bash
   pytest
   pre-commit run --all-files
   ```

5. **推送分支**
   ```bash
   git push origin feature/my-feature
   ```

6. **创建Pull Request**
   - 填写PR模板
   - 关联相关Issue
   - 请求Review

7. **Code Review**
   - 至少需要1个Approve
   - 解决Review意见
   - 确保CI通过

8. **合并**
   - 使用Squash Merge
   - 删除功能分支

### 发版流程

1. 更新版本号（`_version.py`）
2. 更新CHANGELOG.md
3. 创建Release PR
4. 合并后打Tag
5. 自动触发PyPI发布

## 开发新功能指南

### 添加新的模型支持

1. 在`src/agentscope/model/`创建`_xxx_model.py`
2. 继承`ChatModelBase`
3. 实现`__call__`方法
4. 在`__init__.py`中导出

### 添加新的Formatter

1. 在`src/agentscope/formatter/`创建文件
2. 继承`FormatterBase`
3. 实现`format`方法

### 添加新的Memory后端

1. 在`src/agentscope/memory/_working_memory/`创建文件
2. 继承`MemoryBase`
3. 实现`add`, `get_memory`, `delete`, `clear`, `size`方法

### 添加新的工具

1. 在`src/agentscope/tool/`目录下创建工具函数
2. 确保函数有类型注解和docstring
3. 返回`ToolResponse`对象
4. 在`__init__.py`中导出
